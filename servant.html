Servant-suunnittelumalli Pythonissa

Mikä on Servant-malli?

Servant-suunnittelumalli erottaa toiminnallisuuden (palvelun) erilliseen olioon, jota voidaan käyttää monien eri luokkien tai olioiden kanssa ilman, että nämä oliot perivät palvelun toimintaa tai sisältävät sen suoraan. Servant toimii “palveluntarjoajana”, joka suorittaa tehtäviä asiakkaiden puolesta.

Tavoite:
Tarjota yhteinen palvelu useille eri oliolle erillisenä komponenttina ilman perintää tai monimutkaista koodin kopiointia.

Milloin käyttää Servant-mallia?

  • Kun halutaan tarjota sama toiminnallisuus monelle eri luokalle
  • Kun toiminnallisuus ei kuulu olion ydinvastuisiin
  • Kun halutaan välttää moniperintää tai ylimääräistä sidosta olioiden välillä

Esimerkki: Servant tarjoaa tulostuspalvelun

1. Asiakasluokat, joilla on dataa mutta ei tulostustoimintoa

class Document:
    def __init__(self, title, content):
        self.title = title
        self.content = content

class Image:
    def __init__(self, filename, resolution):
        self.filename = filename
        self.resolution = resolution

2. Servant-luokka, joka tarjoaa tulostuspalvelun

class PrintServant:
    def print_document(self, document: Document):
        print(f"Dokumentti: {document.title}\nSisältö: {document.content}")

    def print_image(self, image: Image):
        print(f"Kuva: {image.filename} ({image.resolution} px)")

3. Käyttö

doc = Document("Raportti", "Tämä on raportin sisältö.")
img = Image("kuva.jpg", "1920x1080")

printer = PrintServant()
printer.print_document(doc)
printer.print_image(img)

Tulostus:

Dokumentti: Raportti
Sisältö: Tämä on raportin sisältö.
Kuva: kuva.jpg (1920x1080 px)

Vaihtoehtoinen toteutus: Servant käyttää yhteistä metodia

Jos haluamme tehdä mallista vielä joustavamman, voimme käyttää yhteistä rajapintaa tai attribuutteja, joiden avulla Servant toimii yleisemmin:

class PrintServant:
    def print(self, obj):
        if hasattr(obj, 'title') and hasattr(obj, 'content'):
            print(f"Dokumentti: {obj.title}\nSisältö: {obj.content}")
        elif hasattr(obj, 'filename') and hasattr(obj, 'resolution'):
            print(f"Kuva: {obj.filename} ({obj.resolution} px)")
        else:
            print("Tuntematon objekti, ei voida tulostaa.")

printer = PrintServant()
printer.print(doc)
printer.print(img)

Hyödyt

  • Erotetaan palvelun toiminnallisuus asiakkaan datasta
  • Vähennetään koodin toistoa ja moniperinnän tarvetta
  • Selkeyttää vastuujakoa ja lisää modulaarisuutta

Haitat

  • Servant voi tulla monimutkaiseksi, jos palveluja on paljon
  • Palvelun ja asiakkaan välinen tiivis kytkentä voi kasvaa ilman rajapintoja
  • Ei välttämättä sovi hyvin hyvin heterogeenisiin olioihin ilman lisäsuunnittelua

Yhteenveto

Osa Kuvaus
Asiakkaat Luokat, joilla data tai tila
Servant Erillinen olio, joka tarjoaa palvelun
Vuorovaikutus Servant käyttää asiakkaan dataa ja suorittaa toiminnon

Servant-malli tarjoaa tehokkaan tavan jakaa toiminnallisuutta monille erilaisille olioille ilman perinnän tai monimutkaisen sidonnan tarvetta. Se parantaa koodin selkeyttä ja ylläpidettävyyttä erityisesti laajenevissa järjestelmissä.
