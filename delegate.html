Delegation-suunnittelumalli Pythonissa

Mikä on Delegation Pattern?

Delegation Pattern on suunnittelumalli, jossa olio siirtää tehtävän (vastuun) toiselle oliolle, sen sijaan että toteuttaisi toiminnon itse.

Tarkoitus: Mahdollistaa koodin uudelleenkäyttö, joustava rakenteen muutos ja vastuujen jakaminen ilman perintää.

Delegoinnin periaate

Yksinkertaistettuna:

# Sen sijaan että toteutat metodin itse...
def do_something(self):
    ...

# ...delegate it!
def do_something(self):
    return self._helper.do_something()

Miksi käyttää delegointia?

  • Korvaa perinnän tilanteissa, joissa se ei ole soveltuva
  • Vähentää moniperinnän tarvetta
  • Tukee komposition periaatetta: “käytä koostamista mieluummin kuin perintää”
  • Tekee järjestelmästä joustavamman ja testattavamman

Esimerkki: Delegointi Python-luokkien välillä

1. Palveluluokka

class Printer:
    def print_document(self, content):
        print(f"Tulostetaan: {content}")

2. Asiakasluokka, joka delegoi

class Report:
    def __init__(self, content):
        self.content = content
        self._printer = Printer()  # delegoitu olio

    def print(self):
        # delegointi tapahtuu tässä
        self._printer.print_document(self.content)

3. Käyttö

report = Report("Q3 Talousraportti")
report.print()

Tulostus:

Tulostetaan: Q3 Talousraportti

Esimerkki: Usean vastuun delegointi

class Engine:
    def start(self):
        print("Moottori käynnistyy.")

class Radio:
    def play(self):
        print("Soitetaan musiikkia.")

class Car:
    def __init__(self):
        self.engine = Engine()
        self.radio = Radio()

    def start(self):
        self.engine.start()  # delegointi moottorille

    def play_music(self):
        self.radio.play()    # delegointi radiolle

car = Car()
car.start()        # Moottori käynnistyy.
car.play_music()   # Soitetaan musiikkia.

Delegointi vs Perintä

Ominaisuus Delegointi Perintä
Rakenne “Has-a” (kompositio) “Is-a” (perintä)
Joustavuus ✅ Korkea (voi vaihtaa delegateja) ⚠️ Alhaisempi
Testattavuus ✅ Hyvä ⚠️ Vaatii usein enemmän kontekstia
Uudelleenkäyttö ✅ Voi käyttää useita delegateja ⚠️ Sidottu perintäketjuun

Delegointi dynaamisesti Pythonissa

Pythonin dynaamisuus mahdollistaa myös yleisdelegoinnin __getattr__-menetelmällä:

class Delegator:
    def __init__(self, target):
        self._target = target

    def __getattr__(self, name):
        # delegoi kaikki tuntemattomat metodit targetille
        return getattr(self._target, name)

class RealWorker:
    def work(self):
        print("Työntekijä työskentelee.")

delegator = Delegator(RealWorker())
delegator.work()  # → Työntekijä työskentelee.

Yhteenveto

Ominaisuus Kuvaus
Tarkoitus Siirtää toiminto toiselle oliolle (delegate)
Python-toteutus Käsin metodikutsussa tai __getattr__-metodilla
Hyödyt Joustavuus, uudelleenkäyttö, ei tarvitse perintää
Haitat Voi lisätä välikerroksia, vaikeuttaa debuggausta
Soveltuu Adapterit, wrapperit, palvelut, rajapinnat, komposiitit

Pähkinänkuoressa

class Helper:
    def do(self):
        print("Apuolio tekee työn.")

class Client:
    def __init__(self):
        self.helper = Helper()

    def do(self):
        return self.helper.do()  # delegointi

Delegation Pattern tukee hyvää ohjelmointikäytäntöä: yhden vastuun per peruskomponentti, ja mahdollistaa laajennettavan ja ylläpidettävän rakenteen ilman monimutkaisia perintärakenteita.
