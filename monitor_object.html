Monitor Object -suunnittelumalli Pythonissa

Mikä on Monitor Object -malli?

Monitor Object on suunnittelumalli, jossa objekti hallitsee itsenäisesti säikeiden synkronoinnin siten, että vain yksi säie kerrallaan voi käyttää sen kriittisiä metodeja tai dataa. Toisin sanoen, se kapseloi lukituksen omaan sisäiseen toimintaansa ja tekee säikeiden välisestä yhteentoimivuudesta turvallista ja hallittua.

Tavoite:
Yhdistää jaetun resurssin ja sen synkronoinnin samaan olioon, jolloin monisäikeinen käyttö on turvallista ja hallittua.

Milloin käyttää?

  • Kun haluat kapseloida jaetun tilan ja sen käytön turvallisesti säikeiden kesken
  • Kun haluat estää säikeitä pääsemästä toistensa kriittisiin osiin
  • Kun et halua käyttää “raakaa” lukitusta ulkoisesti vaan antaa olion hoitaa sen itse

Yleinen rakenne

Monitor Object -mallissa:

  • Olio sisältää jaetun datan
  • Olion metodit lukitsevat ja vapauttavat pääsyn automaattisesti
  • Usein käytetään threading.Lock, threading.RLock tai threading.Condition

Esimerkki: Säieturvallinen pankkitili (Monitor Object)

import threading
import time

class BankAccount:
    def __init__(self, balance=0):
        self.balance = balance
        self._lock = threading.Lock()  # lukko on kapseloitu olion sisään

    def deposit(self, amount):
        with self._lock:
            print(f"[{threading.current_thread().name}] Tallettaa {amount}")
            local_balance = self.balance
            time.sleep(0.01)  # Simuloidaan viivettä
            self.balance = local_balance + amount
            print(f"[{threading.current_thread().name}] Uusi saldo: {self.balance}")

    def withdraw(self, amount):
        with self._lock:
            if self.balance >= amount:
                print(f"[{threading.current_thread().name}] Nostaa {amount}")
                self.balance -= amount
                print(f"[{threading.current_thread().name}] Uusi saldo: {self.balance}")
                return True
            else:
                print(f"[{threading.current_thread().name}] Ei tarpeeksi katetta!")
                return False

Käyttö esimerkkinä:

account = BankAccount(100)

def run_transactions():
    account.deposit(50)
    account.withdraw(30)

threads = [threading.Thread(target=run_transactions, name=f"Säie-{i}") for i in range(5)]

for t in threads:
    t.start()
for t in threads:
    t.join()

print(f"\nLopullinen saldo: {account.balance}")

Tulostus (vaihtelee):

[Säie-0] Tallettaa 50
[Säie-0] Uusi saldo: 150
[Säie-0] Nostaa 30
[Säie-0] Uusi saldo: 120
...
Lopullinen saldo: 200

Huom:
Ilman with self._lock, tulos olisi virheellinen race conditionin vuoksi.

Monitor Object vs. Explicit Locking

Ominaisuus Monitor Object Suora lukitus
Lukituslogiikka Piilotettu olion sisälle Käyttäjän vastuulla
Käytön turvallisuus Parempi, virheiden riski pienempi Altis virheille (esim. unohdettu release)
Ylläpidettävyys Helppo käyttää Vaatii tarkkaa hallintaa

Hyödyt

  • Kapseloi synkronoinnin ja tekee siitä läpinäkyvää olion käyttäjälle
  • Estää race conditionit ja muita rinnakkaisuuden virheitä
  • Parantaa modulaarisuutta: synkronointi on osa olion sisäistä rakennetta

Haitat

  • Saattaa piilottaa suorituskykyongelmia, jos lukot estävät liikaa rinnakkaisuutta
  • Ei toimi helposti moniprosessitilanteessa (käytä silloin multiprocessing)
  • Vaikeampi debugata, jos säikeet jäävät odottamaan (esim. deadlock)

Sovelluskohteita

  • Pankkijärjestelmät ja varastointi
  • Säieturvalliset välimuistit
  • Rinnakkaiset laskennalliset objektit
  • Tuottaja–kuluttaja-mallit

Yhteenveto

Monitor Object on tehokas ja selkeä tapa rakentaa säieturvallisia olioita. Sen avulla ohjelmoija voi keskittyä liiketoimintalogiikkaan ja jättää synkronoinnin olion sisäiseksi huoleksi.

Ominaisuus Kuvaus
Synkronointi Rakennetaan olion sisään
Turvallisuus Estää samanaikaisen pääsyn
Rakenne Käyttää Lock, RLock tai Condition
Tavoite Helppo ja turvallinen monisäikeisyys
