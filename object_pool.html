Object Pool -suunnittelumalli Pythonissa

Mikä on Object Pool?

Object Pool (eli olioiden allas) on suunnittelumalli, jossa joukko valmiiksi luotuja olioita pidetään käytettävissä uudelleenkäyttöä varten. Kun olio tarvitaan, se haetaan poolista. Kun sen käyttö on ohi, se palautetaan pooliin, eikä tuhota.

Tarkoitus: Vähentää olioiden luontikustannuksia ja parantaa suorituskykyä uudelleenkäytön avulla.

Käyttökohteita

  • Verkkoyhteydet / socketit
  • Tietokantayhteydet
  • Grafiikkamoottorit / pelit (esim. luotien instanssit)
  • Suorituskykykriittiset järjestelmät
  • Raskaat objektit, kuten PDF-lukijat tai renderöijät

Esimerkki: Yksinkertainen Object Pool Pythonissa

Kuvitellaan, että olioiden luonti kestää jonkin aikaa (esim. yhteys ulkoiseen palveluun).

1. Raskas olio

import time

class ExpensiveObject:
    def __init__(self):
        print("Luodaan kallis olio...")
        time.sleep(1)  # Simuloi raskasta luontia

    def use(self):
        print("Oliota käytetään.")

2. Object Pool -luokka

class ObjectPool:
    def __init__(self, size=2):
        self._available = [ExpensiveObject() for _ in range(size)]
        self._in_use = []

    def acquire(self):
        if not self._available:
            raise Exception("Ei vapaita olioita!")
        obj = self._available.pop()
        self._in_use.append(obj)
        return obj

    def release(self, obj):
        if obj in self._in_use:
            self._in_use.remove(obj)
            self._available.append(obj)

3. Käyttöesimerkki

pool = ObjectPool(size=2)

# Haetaan ensimmäinen olio
obj1 = pool.acquire()
obj1.use()

# Haetaan toinen olio
obj2 = pool.acquire()
obj2.use()

# Palautetaan olio takaisin pooliin
pool.release(obj1)

# Haetaan uudelleen
obj3 = pool.acquire()
obj3.use()

print(obj1 is obj3)  # True – saatiin sama olio

Monisäikeisyys: turvattu Object Pool

Pythonin queue.Queue sopii hyvin säieturvalliseen Object Pool -malliin:

from queue import Queue
from threading import Thread

class ThreadSafePool:
    def __init__(self, size=2):
        self.pool = Queue(maxsize=size)
        for _ in range(size):
            self.pool.put(ExpensiveObject())

    def acquire(self):
        return self.pool.get()

    def release(self, obj):
        self.pool.put(obj)

Käyttö säikeiden kanssa:

def task(pool, name):
    obj = pool.acquire()
    print(f"{name} käyttää oliota")
    time.sleep(0.5)
    pool.release(obj)
    print(f"{name} palautti olion")

pool = ThreadSafePool()

threads = [
    Thread(target=task, args=(pool, f"Säie {i}"))
    for i in range(4)
]

for t in threads:
    t.start()
for t in threads:
    t.join()

Vinkki: Objektin palauttaminen automaattisesti

Voit käyttää context manageria, jotta objektit palautuvat automaattisesti:

from contextlib import contextmanager

@contextmanager
def pooled_object(pool):
    obj = pool.acquire()
    try:
        yield obj
    finally:
        pool.release(obj)

# Käyttö:
with pooled_object(pool) as obj:
    obj.use()

Object Pool on hyödyllinen silloin, kun olioiden luonti on kallista, mutta olioita voidaan kierrättää. Pythonissa tämä onnistuu helposti joko yksinkertaisella listalla tai säieturvallisesti queue.Queue-rakenteella.
