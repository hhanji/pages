Fluent Interface -suunnittelumalli Pythonissa

Mikä on Fluent Interface?

Fluent Interface on suunnittelumalli, jossa metodikutsut on järjestetty ketjutettaviksi kutsuiksi, jotka palauttavat yleensä saman olion (tai sen tilan), jolloin kutsuja voi yhdistää sulavasti peräkkäin.

Tavoite:

  • Parantaa koodin luettavuutta
  • Mahdollistaa konfiguroinnin tai toiminnan rakentaminen “lauseenomaisesti”
  • Vähentää toistuvaa koodia

Perusidea

obj.method1().method2().method3()

Jokainen metodi palauttaa olion (self), jolloin ketju jatkuu.

Esimerkki: Fluent Interface luokka

class QueryBuilder:
    def __init__(self):
        self._query = ""

    def select(self, *fields):
        fields_list = ", ".join(fields)
        self._query += f"SELECT {fields_list} "
        return self

    def from_table(self, table_name):
        self._query += f"FROM {table_name} "
        return self

    def where(self, condition):
        self._query += f"WHERE {condition} "
        return self

    def build(self):
        return self._query.strip()

Käyttö

qb = QueryBuilder()
query = (qb.select("name", "age")
           .from_table("users")
           .where("age > 18")
           .build())

print(query)

Tulostus:

SELECT name, age FROM users WHERE age > 18

Toinen esimerkki: Fluent Interface ketjutettava konfiguraattori

class CarConfigurator:
    def __init__(self):
        self._color = None
        self._engine = None
        self._sunroof = False

    def color(self, color_name):
        self._color = color_name
        return self

    def engine(self, engine_type):
        self._engine = engine_type
        return self

    def sunroof(self, has_sunroof=True):
        self._sunroof = has_sunroof
        return self

    def build(self):
        return f"Auto: Väri={self._color}, Moottori={self._engine}, Kattoluukku={self._sunroof}"

config = CarConfigurator()
car_description = (config.color("Punainen")
                         .engine("V8")
                         .sunroof(True)
                         .build())

print(car_description)

Tulostus:

Auto: Väri=Punainen, Moottori=V8, Kattoluukku=True

Fluent Interface -mallin hyödyt

  • Koodi on selkeää ja helposti luettavaa
  • Toimii hyvin tilanteissa, joissa halutaan ketjuttaa konfiguraatioita tai toimintoja
  • Vähentää koodin määrää ja parantaa sen ylläpidettävyyttä

Huomioitavaa

  • Metodien tulee aina palauttaa olio (self) tai tarvittaessa muu olio, jotta ketjutus toimii
  • Liian pitkät ketjut voivat heikentää luettavuutta, joten tasapaino on tärkeä
  • Ei sovi kaikkiin käyttötapauksiin, mutta erityisen hyödyllinen rakentajissa, kyselyissä ja konfiguraattoreissa

Yhteenveto

Ominaisuus Kuvaus
Tarkoitus Mahdollistaa ketjutettu metodikutsu
Toteutus Palautetaan self metodeista
Käyttökohteet Rakentajat, kyselyt, konfiguraattorit
Edut Luettavuus, tiiviys, intuitiivisuus
Haitat Voi johtaa liian pitkiin ketjuihin

Fluent Interface -malli tekee koodista luonnollisemmin luettavaa ja rakentaa monimutkaisia toimintoja helposti ja selkeästi.
