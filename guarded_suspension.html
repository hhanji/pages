Guarded Suspension -suunnittelumalli Pythonissa

Mikä on Guarded Suspension?

Guarded Suspension -suunnittelumalli perustuu ajatukseen, että jokin operaatio odottaa, kunnes sen suorittamiseen vaadittu ehto täyttyy. Toimintoa ei siis suoriteta heti, vaan se keskeytetään ja jatkuu vasta, kun resurssi tai tila on käyttökelpoinen.

Tämä malli on hyödyllinen erityisesti monisäikeisessä ohjelmoinnissa, missä säikeiden välinen yhteistyö vaatii hallittua odottamista, jotta vältetään kilpailevat tilanteet ja virheelliset tilat.

Tavoite:
Varmistaa, että operaatio suoritetaan vain silloin, kun sen vaatima tila on saavutettu – muuten odotetaan.

Milloin käyttää Guarded Suspension -mallia?

  • Kun säie ei voi suorittaa toimintoa ennen kuin tietyt ehdot täyttyvät
  • Kun resurssia ei ole heti saatavilla ja säikeen pitää odottaa sitä turvallisesti
  • Kun halutaan mallintaa tilaa odottavaa logiikkaa ilman aktiivista kuormitusta (busy waiting)

Esimerkki: Viestien käsittely jonosta vain kun niitä on saatavilla

import threading
import time
from collections import deque

class MessageQueue:
    def __init__(self):
        self.queue = deque()
        self.condition = threading.Condition()

    def put_message(self, message):
        with self.condition:
            self.queue.append(message)
            print(f"[Tuottaja] Viesti lisätty: {message}")
            self.condition.notify()  # Herätä odottava säie

    def get_message(self):
        with self.condition:
            while not self.queue:
                print("[Kuluttaja] Ei viestejä – odotetaan...")
                self.condition.wait()  # Odotetaan, kunnes viesti on saatavilla
            message = self.queue.popleft()
            print(f"[Kuluttaja] Viesti vastaanotettu: {message}")
            return message

Tuottaja ja kuluttaja

def producer(q):
    for i in range(3):
        time.sleep(1)
        q.put_message(f"Viesti {i+1}")

def consumer(q):
    for _ in range(3):
        msg = q.get_message()
        print(f"[Kuluttaja] Käsittelee: {msg}")
        time.sleep(2)

Käyttö

q = MessageQueue()

t1 = threading.Thread(target=consumer, args=(q,))
t2 = threading.Thread(target=producer, args=(q,))

t1.start()
t2.start()

t1.join()
t2.join()

Miten Guarded Suspension toimii tässä?

  • Kuluttaja (consumer) yrittää hakea viestin, mutta pysähtyy odottamaan condition.wait(), jos jono on tyhjä
  • Tuottaja (producer) lisää viestejä ja herättää odottavat säikeet condition.notify()-kutsulla
  • Näin varmistetaan, että get_message() suoritetaan vain kun viesti on saatavilla

Hyödyt

  • Estää virheelliset tai turhat operaatiot väärässä tilassa
  • Mahdollistaa säikeiden turvallisen odottamisen ilman kuorman lisäystä
  • Helppo yhdistää muihin säieturvallisuusmalleihin (esim. Producer-Consumer)

Haitat

  • Vaatii synkronointimekanismeja (lukot, ehdot), jotka voivat johtaa virheisiin (esim. deadlock)
  • Monimutkaisuus kasvaa, jos vartioituja ehtoja on monta tai ne muuttuvat dynaamisesti

Yhteenveto

Osa Kuvaus
Guarded operation Suoritetaan vain, kun vartioitu ehto täyttyy
Condition.wait() Keskeyttää säikeen odottamaan ehtoa
Condition.notify() Herättää säikeen, kun ehto voi täyttyä
Käyttökohde Tuottaja-kuluttaja, resurssin saatavuus

Guarded Suspension -malli auttaa hallitsemaan monisäikeisten ohjelmien tilariippuvuuksia selkeästi ja tehokkaasti. Mallin avulla voidaan kirjoittaa ohjelmia, jotka odottavat järkevästi eivätkä kuormita järjestelmää tarpeettomasti.
