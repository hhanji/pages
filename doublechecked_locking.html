Double-Checked Locking -suunnittelumalli Pythonissa

Mikä on Double-Checked Locking?

Double-Checked Locking (DCL) on monisäikeisessä ohjelmoinnissa käytetty malli, jonka avulla voidaan laiskasti alustaa jaettu resurssi (esimerkiksi singleton-olio) turvallisesti ja tehokkaasti. Mallissa tarkistetaan ensin, onko resurssi jo alustettu ilman lukkoa, ja otetaan lukko käyttöön vain jos resurssi ei ole alustettu – tämän jälkeen tarkistus tehdään vielä uudelleen lukon sisällä ennen alustamista.

Tavoite:
Vähentää lukitsemisen suorituskykyhaittaa samalla säilyttäen säieturvallisuus resurssin alustuksessa.

Rakenne

  1. Tarkista resurssi ilman lukkoa
  2. Jos resurssia ei ole alustettu, lukitse
  3. Tarkista resurssi uudelleen lukon sisällä
  4. Jos se on edelleen alustamaton, alusta resurssi
  5. Palauta resurssi

Käyttökohteet

  • Singleton-olion laiska ja säieturvallinen alustaminen
  • Kalliiden resurssien (esim. tietokantayhteys, välimuisti) hallittu lataaminen
  • Vältetään turhaa lukitsemista, joka voi haitata suorituskykyä

Esimerkki: Singleton-luokka Double-Checked Locking -mallilla

import threading

class Singleton:
    _instance = None
    _lock = threading.Lock()

    def __init__(self):
        # Oletetaan, että alustaminen on kallis operaatio
        print("Alustetaan Singleton-olio...")

    @classmethod
    def get_instance(cls):
        if cls._instance is None:  # Ensimmäinen tarkistus ilman lukkoa
            with cls._lock:
                if cls._instance is None:  # Toinen tarkistus lukon sisällä
                    cls._instance = cls()
        return cls._instance

Käyttö

def create_singleton():
    instance = Singleton.get_instance()
    print(f"Singleton instanssi: {id(instance)}")

threads = []
for _ in range(5):
    t = threading.Thread(target=create_singleton)
    threads.append(t)
    t.start()

for t in threads:
    t.join()

Huomio Pythonissa

  • Pythonin GIL (Global Interpreter Lock) tekee yksinkertaisissa tapauksissa monisäikeisistä lukituksista vähemmän tarpeellisia, mutta Double-Checked Locking voi silti olla hyödyllinen esimerkiksi I/O-sidonnaisessa monisäikeisyydessä tai multiprocessing-tilanteissa.
  • Pythonin threading.Lock() toimii oikein säieturvallisissa tilanteissa, mutta sitä ei pidä käyttää prosessien välillä.

Hyödyt

  • Suorituskyky paranee, koska lukkoa ei tarvitse ottaa joka kerta resurssin hakemiseen
  • Turvallinen resurssin alustaminen useista säikeistä käsin
  • Vähentää turhaa synkronointia

Haitat

  • Kaksinkertainen tarkistus voi johtaa monimutkaisempaan koodiin
  • Joissain kielissä (esim. C++) mallin toteuttaminen vaatii erityistä varovaisuutta muistimallin vuoksi — Pythonissa tilanne on yksinkertaisempi
  • Ei sovi, jos alustuksen on tapahduttava aina (ei laiskaa alustusta)
