Thread-specific Storage -suunnittelumalli Pythonissa

Mikä on Thread-specific Storage?

Thread-specific Storage (TSS) tarkoittaa säikeiden omaa yksilöllistä tilaa, joka on kunkin säikeen käytettävissä. Tämä mahdollistaa esimerkiksi säikeiden välisen tietojen eristämisen ilman, että tietoja tarvitsee välittää eksplisiittisesti funktioiden kautta.

Tavoite:
Tarjota jokaiselle säikeelle oma kopio muuttujasta tai tilasta, jolloin säikeet eivät vahingossa vaikuta toistensa tietoihin.

Milloin käyttää?

  • Kun halutaan säikeille omaa muuttujaa tai tilaa
  • Kun halutaan välttää säikeiden välistä tilan sekaantumista
  • Kun halutaan välttää monimutkaista lukitusten hallintaa yksinkertaisissa tilanteissa

Pythonin threading.local -luokka

Python tarjoaa threading.local-luokan, jonka avulla voi luoda säikeille oman tilan.

Esimerkki: Yksinkertainen Thread-specific Storage

import threading

# Luodaan thread-local storage -objekti
thread_local = threading.local()

def worker(number):
    thread_local.value = number
    print(f"Säie {number} asetettu arvoon {thread_local.value}")
    process()

def process():
    # Käytetään säikeen omaa arvoa
    print(f"Säie {threading.current_thread().name} käsittelee arvoa {thread_local.value}")

threads = []

for i in range(3):
    t = threading.Thread(target=worker, args=(i,))
    threads.append(t)
    t.start()

for t in threads:
    t.join()

Selitys:

  • thread_local.value on jokaiselle säikeelle oma muuttuja
  • Säikeet eivät jaa toistensa value-arvoa, vaikka muuttuja näyttää samalta

Esimerkki: Thread-specific Storage käytössä tietokantayhteyksissä

import threading

class DatabaseConnection:
    def __init__(self):
        self.connected = False

    def connect(self):
        print(f"{threading.current_thread().name} yhdistää tietokantaan.")
        self.connected = True

    def query(self, sql):
        if not self.connected:
            raise Exception("Ei yhteyttä tietokantaan!")
        print(f"{threading.current_thread().name} suorittaa kyselyn: {sql}")

thread_local = threading.local()

def get_connection():
    if not hasattr(thread_local, "db_conn"):
        thread_local.db_conn = DatabaseConnection()
        thread_local.db_conn.connect()
    return thread_local.db_conn

def worker(sql):
    conn = get_connection()
    conn.query(sql)

threads = []
queries = ["SELECT * FROM users", "SELECT * FROM orders", "SELECT * FROM products"]

for q in queries:
    t = threading.Thread(target=worker, args=(q,))
    threads.append(t)
    t.start()

for t in threads:
    t.join()

Hyödyt

  • Säikeiden välinen tila pysyy erillään ja turvallisena
  • Ei tarvita monimutkaisia lukkoja yksinkertaisissa tapauksissa
  • Selkeä ja helppokäyttöinen rajapinta säikeiden tilalle

Haasteet

  • Ei sovi monisäikeisten objektien jakamiseen, vain säikeiden omaan tilaan
  • Liiallinen käyttö voi johtaa vaikeasti jäljitettäviin virheisiin
  • Tarvitaan huolellisuutta, ettei säikeille jää “vanhoja” arvoja

Yhteenveto

Thread-specific Storage on tehokas malli, joka mahdollistaa säikeiden oman tilan hallinnan Pythonissa helposti threading.local-luokan avulla. Se auttaa tekemään säikeistä turvallisempia ja eristetympiä.
