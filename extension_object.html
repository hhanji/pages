Extension Object -suunnittelumalli Pythonissa

Mikä on Extension Object -malli?

Extension Object -malli mahdollistaa olion toiminnallisuuden laajentamisen kiinnittämällä siihen erillisiä laajennuksia (extensions), jotka tarjoavat lisäominaisuuksia.

Tarkoitus:

  • Mahdollistaa olion laajentaminen ilman, että sen koodia muutetaan.
  • Eristää laajennukset omiksi erillisiksi objekteiksi, jotka voidaan liittää dynaamisesti.
  • Tarjoaa joustavuutta ja modulaarisuutta.

Mihin mallia käytetään?

  • Laajennettavat järjestelmät, joissa olion toiminnallisuus voi muuttua tai kasvaa ajan myötä
  • Plug-in -tyyppiset arkkitehtuurit
  • Toiminnallisuuden konfigurointi ajo-ajossa

Python-esimerkki: Extension Object -malli

1. Perusluokka, johon laajennukset kiinnitetään

class CoreObject:
    def __init__(self):
        self._extensions = {}

    def add_extension(self, name, extension):
        self._extensions[name] = extension

    def get_extension(self, name):
        return self._extensions.get(name, None)

2. Laajennusluokat (extensionit)

class LoggerExtension:
    def log(self, message):
        print(f"[LOG]: {message}")

class SerializerExtension:
    def serialize(self, data):
        import json
        return json.dumps(data)

3. Käyttö: lisätään ja käytetään laajennuksia

obj = CoreObject()

# Lisätään logger-laajennus
obj.add_extension('logger', LoggerExtension())

# Lisätään serializer-laajennus
obj.add_extension('serializer', SerializerExtension())

# Käytetään laajennuksia
logger = obj.get_extension('logger')
if logger:
    logger.log("Tämä on logiviesti.")

serializer = obj.get_extension('serializer')
if serializer:
    data = {'name': 'Maija', 'age': 30}
    print(serializer.serialize(data))

Tulostus:

[LOG]: Tämä on logiviesti.
{"name": "Maija", "age": 30}

4. Vapaaehtoinen dynaaminen delegointi laajennuksille

Voimme myös tehdä perusolion metodin, joka delegoi laajennuksille, mikä helpottaa käyttöä:

class CoreObject:
    def __init__(self):
        self._extensions = {}

    def add_extension(self, name, extension):
        self._extensions[name] = extension

    def __getattr__(self, attr):
        for ext in self._extensions.values():
            if hasattr(ext, attr):
                return getattr(ext, attr)
        raise AttributeError(f"'{self.__class__.__name__}' object has no attribute '{attr}'")

obj = CoreObject()
obj.add_extension('logger', LoggerExtension())
obj.log("Delegoitua lokia ilman get_extension-kutsua.")

Tulostus:

[LOG]: Delegoitua lokia ilman get_extension-kutsua.

Hyödyt

✅ Laajennukset erillisinä, modulaarisina kokonaisuuksina
✅ Voidaan lisätä, poistaa tai vaihtaa ajo-ajossa
✅ Vähentää perintäketjujen monimutkaisuutta
✅ Mahdollistaa olion dynaamisen toiminnallisuuden laajentamisen

Haitat

⚠️ Lisää hieman monimutkaisuutta koodiin
⚠️ Vaatii hallintaa laajennusten nimistä ja vastuista
⚠️ Voi vaikeuttaa virheiden jäljittämistä, jos delegointi on liian dynaamista
