Marker-suunnittelumalli Pythonissa

Mikä on Marker Pattern?

Marker Pattern on suunnittelumalli, jossa luokka “merkitään” kuuluvaksi tiettyyn ryhmään tai omaavaksi tietyn ominaisuuden ilman metodien tai logiikan lisäämistä.

Tarkoitus: Mahdollistaa, että olioita voidaan käsitellä eri tavalla pelkän luokkatiedon perusteella, ilman että ne toteuttavat konkreettista toiminnallisuutta.

Marker-mallin käyttötarkoituksia

Käyttökohde Esimerkki
Turvallisuus Merkitään luokka turvalliseksi ajaa (esim. sandbox)
Sarjallistus Vain tietyt luokat saavat olla serialisoitavissa
Erikoiskohtelu Suodatetaan tai käsitellään erityisellä tavalla
Validointi Vain tietyt luokat sallitaan syötteeksi

Marker-malli vs muut mallit

  • Ei sisällä metodeja → ei toiminnallisuutta
  • Käyttää perintää tai luokkien tunnistamista
  • Ei riko YAGNI- tai KISS-periaatteita (keep it simple)

Esimerkki: Sarjallistettavat oliot (Serializable)

1. Marker-luokka

class Serializable:
    """Marker-luokka – ei sisällä mitään."""
    pass

2. Muita luokkia

class User(Serializable):
    def __init__(self, name):
        self.name = name

class Logger:
    def __init__(self, level):
        self.level = level

3. Serialisointitoiminto

import json

def serialize(obj):
    if isinstance(obj, Serializable):
        return json.dumps(obj.__dict__)
    else:
        raise TypeError("Olio ei ole serialisoitava.")

4. Käyttö

user = User("Matti")
data = serialize(user)
print(data)  # {"name": "Matti"}

logger = Logger("INFO")
serialize(logger)  # TypeError: Olio ei ole serialisoitava

Vaihtoehto: Marker-dekoraattori Pythonissa

Koska Python on dynaaminen kieli, voit käyttää myös dekorointia merkitsemiseen:

def serializable(cls):
    cls._is_serializable = True
    return cls

@serializable
class Product:
    def __init__(self, name):
        self.name = name

def serialize(obj):
    if getattr(obj.__class__, '_is_serializable', False):
        return json.dumps(obj.__dict__)
    else:
        raise TypeError("Olio ei ole serialisoitava.")

Hyödyt

✅ Erittäin kevyt ja yksinkertainen
✅ Ei sido luokkaa ylimääräiseen toiminnallisuuteen
✅ Toimii hyvin tyypityksen ja introspektion kanssa
✅ Pythonissa helppo laajentaa myös dekoraattorilla

Haitat

⚠️ Ei tarjoa mitään toiminnallista API\:a
⚠️ Voi olla vaikea hahmottaa, jos luokilla ei ole näkyvää sisältöä
⚠️ Ei suojaa väärinkäytöltä, jos luokkaa “merkitään” ilman tarkoitusta

Yhteenveto

Ominaisuus Selitys
Tarkoitus Merkitä luokat tiettyihin ryhmiin ilman toiminnallisuutta
Pythonin toteutus Perintä, tyhjät luokat tai dekoraattorit
Esimerkkejä Serialisointi, validointi, turvallisuus, käyttöliittymät
Hyödyllinen kun Haluat eriyttää tiettyjen olioiden käsittelyn kevyesti

Esimerkki yhdellä silmäyksellä

class Marker:
    pass

class MyClass(Marker):
    pass

obj = MyClass()

if isinstance(obj, Marker):
    print("Tämä olio on merkitty!")

Marker Pattern on yksinkertainen mutta tehokas työkalu olioiden luokittelemiseen tai erityiskohteluun ilman raskasta rakenteellista muutosta. Pythonissa se on helppo toteuttaa perinnän tai jopa yksinkertaisten attribuuttien avulla.
