Thread Pool -suunnittelumalli Pythonissa

Mikä on Thread Pool?

Thread Pool (säikeiden allas) on suunnittelumalli, jossa etukäteen luodaan joukko säikeitä, joita käytetään tehtävien suorittamiseen. Tämä estää uusien säikeiden jatkuvan luomisen ja tuhoamisen, mikä parantaa suorituskykyä ja resurssien hallintaa.

Tavoite:
Hallita säikeiden elinkaarta tehokkaasti ja mahdollistaa tehtävien rinnakkainen suoritus ilman liiallista resurssien kulutusta.

Milloin käyttää?

  • Kun sovellus suorittaa paljon lyhyitä ja usein toistuvia tehtäviä
  • Kun halutaan rajoittaa samanaikaisten säikeiden määrää
  • Kun halutaan parantaa suorituskykyä ja resurssien käyttöä

Pythonin concurrent.futures.ThreadPoolExecutor

Python tarjoaa valmiin ThreadPoolExecutor-luokan concurrent.futures-moduulissa, joka helpottaa thread poolin käyttöä.

Esimerkki: Yksinkertainen Thread Pool

from concurrent.futures import ThreadPoolExecutor
import time

def task(name):
    print(f"Tehtävä {name} alkaa")
    time.sleep(2)
    print(f"Tehtävä {name} päättyy")
    return f"Tulos {name}"

# Luodaan thread pool, jossa 3 säiettä
with ThreadPoolExecutor(max_workers=3) as executor:
    futures = [executor.submit(task, i) for i in range(5)]

    for future in futures:
        print(future.result())

Selitys:

  • max_workers=3 rajoittaa samaan aikaan käynnissä olevien säikeiden määrän kolmeen
  • submit() käynnistää tehtävän säikeessä
  • future.result() odottaa tehtävän valmistumista ja palauttaa tuloksen

Esimerkki: Thread Pool tiedostojen käsittelyssä

from concurrent.futures import ThreadPoolExecutor
import time

def process_file(filename):
    print(f"Käsitellään tiedostoa {filename}")
    time.sleep(1)  # Simuloidaan käsittelyä
    print(f"Tiedosto {filename} käsitelty")
    return filename.upper()

files = ['tiedosto1.txt', 'tiedosto2.txt', 'tiedosto3.txt', 'tiedosto4.txt']

with ThreadPoolExecutor(max_workers=2) as executor:
    results = list(executor.map(process_file, files))

print("Kaikki tiedostot käsitelty:", results)

Hyödyt

✅ Tehtävien suoritus rinnakkain ilman säikeiden jatkuvaa luomista
✅ Resurssien tehokas hallinta
✅ Helppo käyttää valmiita kirjastoja kuten concurrent.futures
✅ Soveltuu hyvin IO-sidonnaisiin tehtäviin

Haitat ja rajoitukset

⚠️ Pythonin GIL (Global Interpreter Lock) rajoittaa CPU-intensiivisten tehtävien rinnakkaisuutta säikeissä
⚠️ Vaatii huolellista virheenkäsittelyä säikeissä
⚠️ Voi johtaa resurssien ylikuormitukseen, jos pool on liian suuri

Yhteenveto

Thread Pool on tehokas tapa hallita useita säikeitä Pythonissa, erityisesti IO-sidonnaisissa tehtävissä. Pythonin ThreadPoolExecutor tarjoaa helpon tavan hyödyntää tätä mallia.
