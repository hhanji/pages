Service Handler -suunnittelumalli Pythonissa

Mikä on Service Handler?

Service Handler on suunnittelumalli, joka kapseloi palvelujen kutsun, hallinnan ja toteutuksen yhteen paikkaan. Se tarjoaa keskitetyn tavan hallita erilaisia palveluita tai toimintoja, joiden käyttöönotto ja käyttö voidaan pitää erillään asiakaslogiikasta.

Tavoite:
Erotella palveluiden käyttö ja niiden toteutus, helpottaa palveluiden vaihtamista ja laajentamista.

Milloin käyttää?

  • Kun järjestelmässä on useita erilaisia palveluita tai toimintoja
  • Kun halutaan keskitetty palveluiden hallinta
  • Kun palveluiden kutsut halutaan eristää asiakaslogiikasta ja helpottaa ylläpitoa

Yksinkertainen Service Handler -malli Pythonissa

class ServiceHandler:
    def __init__(self):
        self._services = {}

    def register_service(self, name, service):
        self._services[name] = service

    def unregister_service(self, name):
        if name in self._services:
            del self._services[name]

    def execute(self, name, *args, **kwargs):
        service = self._services.get(name)
        if not service:
            raise ValueError(f"Palvelua '{name}' ei löydy.")
        return service(*args, **kwargs)

# Palveluita esimerkkinä
def greeting_service(name):
    return f"Hei, {name}!"

def addition_service(a, b):
    return a + b

# Käyttö
handler = ServiceHandler()
handler.register_service("greet", greeting_service)
handler.register_service("add", addition_service)

print(handler.execute("greet", "Maija"))  # Hei, Maija!
print(handler.execute("add", 5, 7))       # 12

Palveluiden määrittely luokkina ja hallinta

Voidaan toteuttaa myös palvelut luokkina, jolloin ne voivat sisältää tilaa tai monimutkaisempaa logiikkaa.

class GreetingService:
    def greet(self, name):
        return f"Tervehdys, {name}!"

class MathService:
    def add(self, a, b):
        return a + b

class ServiceHandler:
    def __init__(self):
        self._services = {}

    def register_service(self, name, service):
        self._services[name] = service

    def execute(self, name, method, *args, **kwargs):
        service = self._services.get(name)
        if not service:
            raise ValueError(f"Palvelua '{name}' ei löydy.")
        func = getattr(service, method, None)
        if not func:
            raise ValueError(f"Metodia '{method}' ei löydy palvelusta '{name}'.")
        return func(*args, **kwargs)

# Käyttö
handler = ServiceHandler()
handler.register_service("greeting", GreetingService())
handler.register_service("math", MathService())

print(handler.execute("greeting", "greet", "Matti"))  # Tervehdys, Matti!
print(handler.execute("math", "add", 3, 4))           # 7

Hyödyt

  • Keskitetty hallinta palveluille
  • Helppo laajentaa lisäämällä uusia palveluita
  • Selkeä erottelu asiakaslogiikan ja palveluiden välillä
  • Mahdollistaa palveluiden dynaamisen rekisteröinnin ja poiston

Haasteet

  • Voi lisätä pienen ylikuormituksen yksinkertaisiin sovelluksiin
  • Palveluiden metodikutsujen virheenkäsittely vaatii huolellisuutta
  • Palveluiden hallinnan logiikka voi kasvaa monimutkaiseksi suurissa järjestelmissä

Yhteenveto

Service Handler on kätevä malli palveluiden keskitettyyn hallintaan ja käyttöön. Pythonissa se on helppo toteuttaa dynaamisesti, mikä tekee siitä joustavan ja laajennettavan.
