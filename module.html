Module-suunnittelumalli Pythonissa

Mikä on Module Pattern?

Module-suunnittelumalli (engl. Module Pattern) on rakenne, jossa koodi jaetaan erillisiin loogisiin kokonaisuuksiin, joilla on omat vastuunsa ja rajapinnat. Näitä kutsutaan moduleiksi (moduuleiksi), ja ne voivat piilottaa sisäisen toteutuksen tarjoten vain rajatun käyttöliittymän ulkopuoliselle koodille.

Tarkoitus: Piilottaa toteutuksen yksityiskohdat ja suojata globaalia nimialuetta. Mahdollistaa selkeän kapseloinnin, uudelleenkäytön ja selkeyden.

Module-suunnittelumallin ominaisuudet

  • Koodi on jaettu itsenäisiin yksiköihin (moduuleihin)
  • Jokainen moduuli piilottaa sisäisen toteutuksensa
  • Vain halutut toiminnot paljastetaan ulkomaailmalle
  • Ei tarvetta olioille tai luokille – toimii myös funktionaalisesti

Module-suunnittelumalli Pythonissa

Pythonissa moduuli on yksinkertaisesti .py-tiedosto, joka voi sisältää:

  • muuttujia
  • funktioita
  • luokkia
  • ali-moduuleja

Esimerkki: math_utils.py – matemaattinen moduuli

1. Luodaan moduuli (math_utils.py)

# math_utils.py

PI = 3.14159

def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

def _secret_function():
    print("Tätä ei ole tarkoitettu käytettäväksi ulkopuolelta.")

Huom: Funktio _secret_function() on tarkoitettu sisäiseen käyttöön (prefixi on konventio, ei tekninen esto).

2. Käytetään moduulia

# main.py

import math_utils

print(math_utils.add(10, 5))         # 15
print(math_utils.subtract(10, 3))    # 7
print(math_utils.PI)                 # 3.14159

# math_utils._secret_function()     # Mahdollista, mutta ei suositeltavaa

3. Rajauksen tarkempi hallinta: __all__

Voit ohjata, mitä from module import * tuo käyttöön määrittelemällä __all__:

# math_utils.py

__all__ = ['add', 'subtract']

# Nyt `from math_utils import *` tuo vain nämä kaksi funktiota

Esimerkki: Konfiguraatiomoduuli

# config.py

DEBUG = True
DATABASE_URI = "sqlite:///example.db"

def print_config():
    print(f"Debug: {DEBUG}, DB: {DATABASE_URI}")

# main.py

import config

if config.DEBUG:
    config.print_config()

Module Pattern ilman tiedostojakoa

Pythonissa voi toteuttaa moduulin myös sisäisesti yhdellä tiedostolla:

def create_logger():
    _log_level = "INFO"  # Yksityinen muuttuja

    def log(message):
        print(f"[{_log_level}] {message}")

    return log

logger = create_logger()
logger("Sovellus käynnistyi")

Tässä create_logger() toimii moduulin tavoin: se kapseloi yksityisiä tietoja (kuten _log_level).

Module Pattern vs muut mallit

Ominaisuus Module Pattern Singleton / Class-based
Tila Säilyy moduulin sisällä Säilyy olion tai luokan sisällä
Kapselointi Tiedostopohjainen, funktionaalinen Luokkapohjainen
Laajennettavuus Yksinkertainen, selkeä Vaatii olioarkkitehtuuria
Luonnollinen Pythonissa ✅ Kyllä ⚠️ Ei automaattisesti

Hyödyt

✅ Luonnollinen ja sisäänrakennettu Pythonissa
✅ Selkeä koodin jako ja kapselointi
✅ Vähentää globaaleja muuttujia ja riippuvuuksia
✅ Toimii hyvin pienissä ja suurissa projekteissa
✅ Mahdollistaa testauksen ja uudelleenkäytön

Haitat

⚠️ Ei varsinaista suojausta yksityisiltä jäseniltä
⚠️ Ylikäyttö voi johtaa monimutkaiseen riippuvuusverkkoon
⚠️ Ei objektiorientoitunut itsessään (mutta yhdistettävissä muihin malleihin)

Mini-esimerkki yhdellä silmäyksellä

# utils.py
def greet(name):
    return f"Hei, {name}!"

# main.py
import utils
print(utils.greet("Maija"))

Module-suunnittelumalli on yksi tärkeimmistä arkkitehtuurillisista työkaluista Pythonissa. Sen avulla voit pitää koodisi siistinä, uudelleenkäytettävänä ja hallittavana – pienistä skripteistä suurten järjestelmien arkkitehtuuriin.
